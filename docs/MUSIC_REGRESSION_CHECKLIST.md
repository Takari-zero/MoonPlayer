# 音乐模块回归检查清单

本文档用于记录音乐模块当前已确认的关键路径，后续继续开发音乐功能前应先阅读并按需回归，避免把已经稳定的播放队列、播放页返回、A-B、定时器、收藏等能力改回去。

## 一、音乐主页

- 我喜欢入口应进入真实收藏歌曲列表，不使用假数据。
- 最近播放入口应进入真实最近播放列表，不使用假数据。
- 音乐文件夹入口应显示真实扫描到的有音乐文件夹。
- 音乐文件夹点击后在音乐主页内联显示，不跳转单独详情页。
- 音乐文件夹内联标题应显示为 `音乐文件夹-文件夹名`。
- 音乐文件夹内联页应提供“返回音乐文件夹”入口。
- 底部迷你播放器出现时，列表底部必须留出避让空间，最后一项不能被遮挡。

## 二、播放队列

- 进入我喜欢后，播放队列范围应按我喜欢列表生成。
- 进入最近播放后，播放队列范围应按最近播放列表生成。
- 进入 `音乐文件夹-具体文件夹` 后，播放队列范围应按当前文件夹歌曲生成。
- 当前歌曲在当前列表内时，点击“播放全部”不应跳到第一首，也不应重头播放。
- 当前歌曲不在当前列表内时，点击“播放全部”可以从当前列表第一首开始播放。
- 随机播放是用户主动操作，可以立即随机选择当前列表内歌曲开始播放。
- 上一曲 / 下一曲必须只在当前队列范围内工作，不能混入其它列表或其它文件夹歌曲。

## 三、播放页返回

- 从我喜欢进入音乐播放页，返回后仍应显示我喜欢列表。
- 从最近播放进入音乐播放页，返回后仍应显示最近播放列表。
- 从 `音乐文件夹-mp3` 进入音乐播放页，返回后仍应显示 `音乐文件夹-mp3`。
- 从迷你播放器进入音乐播放页，返回后也应保持进入前的音乐子页面。
- 播放页关闭动画露出的底层背景不能固定为音乐主页默认页，应保持进入播放页前的音乐页面状态。

## 四、播放模式

- 播放中切换顺序 / 随机模式，不应重头播放当前歌曲。
- 播放中切换模式，不应切到第一首。
- 暂停中切换模式，不应自动播放。
- 无歌曲时切换模式，只保存模式，不应自动播放。
- 播放模式需要持久记忆，重启后仍使用上次选择的模式，但不自动播放。

## 五、A-B 和定时器

- A-B 第一次点击应设置 A 点。
- A-B 第二次点击应设置 B 点并开启循环。
- A-B 循环到 B 点后应自动 seek 回 A 点并继续播放。
- A-B 第三次点击应关闭循环并清除 A / B 点。
- 手动切歌应清除 A-B 状态。
- “播完当前歌曲后暂停”和“定时后播完整首再暂停”不能与 A-B 同时保持，避免 A-B 阻止歌曲到达结尾。
- 普通倒计时暂停可以和 A-B 共存，到时间后直接暂停播放。
- 定时器面板应能正常打开、确认、关闭和取消定时。

## 六、收藏 / 最近播放

- 播放页收藏按钮轻提示应正确：
  - 加入收藏：`已加入我喜欢`
  - 取消收藏：`已取消喜欢`
- 收藏状态应同步到我喜欢列表。
- 取消收藏后，我喜欢列表应同步移除该歌曲。
- 当前歌曲切换后，播放页收藏状态应自动刷新。
- 最近播放应持久化，重启后仍能恢复最近播放列表。

## 七、推荐测试命令

编译：

```powershell
.\gradlew.bat :app:assembleDebug --console=plain
```

安装与启动：

```powershell
$adb="E:\Android\platform-tools\adb.exe"

& $adb -P 62001 install -r .\app\build\outputs\apk\debug\app-debug.apk
& $adb -P 62001 shell monkey -p com.shenghui.localvibe 1
```

logcat 检查：

```powershell
& $adb -P 62001 logcat -d | Select-String -Pattern "Process: com.shenghui.localvibe|FATAL EXCEPTION|ANR in com.shenghui.localvibe|AndroidRuntime|NullPointerException|IllegalStateException|SecurityException|MainActivity|AudioLibraryScreen|AudioPlayerScreen|Player" -Context 3,3
```

说明：

- 只有 `com.shenghui.localvibe` 进程崩溃才算 App 崩溃。
- MIUI / DynamicIsland / SettingsProvider / WindowManager 等系统噪声不要误判为 App 崩溃。
- 删除、移除、收藏、A-B、定时器、播放队列、播放页返回等真机行为仍需要用户最终确认。
